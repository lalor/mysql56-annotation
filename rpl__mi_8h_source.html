00032 /*****************************************************************************
00033   Replication IO Thread
00034 
00035   Master_info contains:
00036     - information about how to connect to a master
00037     - current master log name
00038     - current master log offset
00039     - misc control variables
00040 
00041   Master_info is initialized once from the master.info repository if such
00042   exists. Otherwise, data members corresponding to master.info fields
00043   are initialized with defaults specified by master-* options. The
00044   initialization is done through mi_init_info() call.
00045 
00046   Logically, the format of master.info repository is presented as follows:
00047 
00048   log_name
00049   log_pos
00050   master_host
00051   master_user
00052   master_pass
00053   master_port
00054   master_connect_retry
00055 
00056   To write out the contents of master.info to disk a call to flush_info()
00057   is required. Currently, it is needed every time we read and queue data
00058   from the master.
00059 
00060   To clean up, call end_info()
00061 
00062 *****************************************************************************/
